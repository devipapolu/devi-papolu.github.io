Visitor Management System (VMS) Deployment with WhatsApp API Integration

Led the deployment of a Visitor Management System (VMS) with WhatsApp API integration on an Ubuntu server (IP: 192.168.1.27). The backend, built with Node.js and Express.js, provides a robust API, while the frontend, developed in React.js, ensures a seamless and responsive user experience. This system enables real-time visitor notifications via the WhatsApp API and ensures high availability through PM2.

  • Technologies Used: Node.js, Express.js, React.js, MongoDB, Ubuntu Linux, PM2, Serve, OpenSSL, UFW, WhatsApp API.
  • Key Components: Deployment on Ubuntu server, WhatsApp API for real-time notifications, SSL configuration for secure communication, process management with PM2, firewall configuration with UFW, and performance monitoring.

Deployment Process Overview:

  1. Server Setup & Application Directory: Accessed the server via SSH (ssh root@192.168.1.27) and navigated to the project directory (cd /root/vms-with-whatsappapi/vms-finalfrontend-withapi) to manage deployment and set up necessary files.
  2. Dependency Installation & Building: Ran npm install to install required dependencies for both backend and frontend, then built the React app using npm run build.
  3. Serving the App with SSL: Installed Serve (npm install -g serve) to serve the React build folder, and used OpenSSL to generate a self-signed SSL certificate for secure HTTPS communication.
  4. Firewall Configuration: Configured the firewall with UFW, allowing traffic through port 3000 for app accessibility while maintaining server security.
  5. Process Management with PM2: Ensured the app ran continuously by starting it with PM2, enabling automatic restarts upon server reboots.
